如果你的目标是**真正读懂 Spring，而不是只会用 Spring**，我并不建议从 `BeanFactory` 或 `ApplicationContext` 开始逐行看源码。Spring 的代码量超过百万行，而且模块之间依赖非常复杂，直接阅读很容易迷失。

**最好的方式是：以测试用例(Test)驱动源码阅读。**

Spring 官方自己的测试，就是最好的"教程"。

---

## 如果你的目标是“精通 Spring”，我还建议增加一条“按 `refresh()` 主线阅读”的路线

Spring 容器的启动几乎都围绕 `AbstractApplicationContext.refresh()` 展开。可以把它作为贯穿全程的主线，每读完一个模块，就回到 `refresh()` 看对应步骤：

```
refresh()
 ├── prepareRefresh()
 ├── obtainFreshBeanFactory()
 ├── prepareBeanFactory()
 ├── postProcessBeanFactory()
 ├── invokeBeanFactoryPostProcessors()   ← @Configuration、BeanDefinition 扩展
 ├── registerBeanPostProcessors()        ← BeanPostProcessor 注册
 ├── initMessageSource()
 ├── initApplicationEventMulticaster()   ← 事件机制
 ├── onRefresh()
 ├── registerListeners()
 ├── finishBeanFactoryInitialization()   ← 实例化单例 Bean（IOC 核心）
 └── finishRefresh()                     ← 发布容器刷新完成事件
```

这样阅读的好处是能够把 **IOC、DI、Bean 生命周期、事件机制、AOP、事务** 都串联到同一条启动流程中，而不是割裂地理解各个模块。这也是阅读 Spring Framework 源码时最常见、最高效的方法。
